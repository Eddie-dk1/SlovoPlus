# 🚀 MVP-план проекта “Слово+”

### 🎯 Цель MVP

Сделать **рабочий, красивый сайт**, где:

* можно ввести слово

Для MVP я бы сделал **4 основные страницы**.

* примеры

* рекомендации по уместности
* возможные ошибки
* похожие слова

### Задача страницы:

Показать полный разбор слова.

---

Раздел для изучения лексики и культуры речи.

### Что на ней:

  * часто путаемые слова

Показать, что сайт — это не только поиск, но и образовательная платформа.

## Что хранить в `types/word.ts`

Нужен единый тип слова, чтобы данные были аккуратными.

Пример структуры:

```ts
export interface WordData {
  word: string;
  definition: string;
  simpleExplanation?: string;
  partOfSpeech?: string;
  examples?: string[];
  style?: string[];
  usageTips?: string[];
  mistakes?: string[];
  relatedWords?: string[];
}
```

## Как связать всё между собой

### Сценарий:

### 1. На главной:

`SearchBar` принимает слово

### 2. После поиска:

переход на страницу
`/word/ирония`

### 3. Страница `WordPage`

* получает параметр из URL
* вызывает `dictionaryApi.ts`
* получает данные
* дополняет локальными данными:

  * стиль
  * ошибки
  * рекомендации
  * похожие слова

### 4. Отрисовывает:

* `WordCard`
* `DefinitionBlock`
* `ExamplesBlock`
* `StyleBadge`
* `UsageTips`
* `RelatedWords`
